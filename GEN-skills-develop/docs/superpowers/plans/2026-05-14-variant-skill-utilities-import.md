# Variant Skill Utilities Import Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Import the four `feat/ark` variant-specific skills as reusable utilities and wire them into existing variant runtime requirements.

**Architecture:** Use the existing `utility` package model and existing `variant.requiresUtilities` resolver. Keep provider capabilities and installer architecture unchanged. Because utility manifests declare `skill.source: skill`, place imported support files under each utility package's `skill/` folder so registry archives and installed skill folders include the referenced assets.

**Tech Stack:** TypeScript, Vitest, YAML manifests, tar registry archives, Markdown skill bodies.

---
