# Frontend Variant: UI Feature Implementation

This template extends shared development guidance with frontend behavior rules.

## Rules
- Separate view rendering from side effects.
- Keep state transitions explicit and testable.
- Handle loading, empty, success, and error states deterministically.
- Preserve accessibility behavior for keyboard and screen reader usage.
- Avoid coupling UI components directly to transport clients.

## Typical Checks
- state transition coverage
- interaction and accessibility checks
- resilient rendering under partial data
